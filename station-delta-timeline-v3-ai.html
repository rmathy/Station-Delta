Fix 2 save key collisions across officer sims
Two pairs of sims share the same localStorage save key. If a student plays both in the same browser, one silently corrupts the other's progress. This is a two-line fix per file but it will cause real user support problems if it ships to paying customers.
Chaos Officer + Comms Officer
Both use: vco_v1
→ Change Chaos to: vcho_v1
→ Change Comms to: vcmo_v1
Deployment Officer + Data Officer
Both use: vdo_v1
→ Change Deployment to: vdep_v1
→ Change Data to: vdat_v1
One-line fix per file. Open each file, search for SAVE_KEY, update the string. Commit all four changes at once. Done in under 10 minutes.

Context — read before starting Phase 3
Why move from HTML files to a real app — and what that means for a solo founder
This phase converts Station Delta from 13 independent HTML files into a proper SaaS product. This is the right move, and it needs to happen before you have paying fleet customers — not after. The specific problems it solves:
What gets fixed in this phase
1. Real execution score — Commands that actually run via WebContainers. Real ls, cat, git, bun install. Not simulated output.

2. API key safety — Key lives only on the server. A leaked endpoint requires a valid session token to use. Your key is never in client JavaScript.

3. Fleet is possible — Progress in a database means manager dashboards, CLI heat maps, time-to-solution tracking, and custom scenarios per org.

4. Save key collisions die forever — localStorage goes away. Every user has a real session.
For a solo non-technical founder: Claude writes every line of this code. Your job is to understand what each piece does well enough to paste it in the right place, set the right environment variables, and test that it works. Each step below is scoped to be completable in 1–2 days with AI assistance.
