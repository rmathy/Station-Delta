# Station Delta 9.5 Gate Rubric
### Version 2.0 — Updated with Accessibility, Cross-Device, and Analytics Standards

Use this rubric before merge for any simulation that is marked "live" or release-candidate.

---

## Gate Rule
- A sim must score **9.5 or higher overall** to pass.
- No category may score below **9.0**.
- Any **Critical fail** in command acceptance, accessibility blocking, or learner-blocking UX = automatic fail until fixed.

## Scoring Method
- Each category is scored 0.0 to 10.0.
- Overall score = weighted average.
- Record evidence with file references and test notes in the release PR.

## Categories And Weights
1. UX Clarity And Flow — 18%
2. Pedagogy Compliance — 23%
3. Command Acceptance Robustness — 23%
4. Performance And Stability — 13%
5. Narrative And Voice Quality — 13%
6. Accessibility And Inclusion — 10% *(new)*

> **Weights total 100%. Categories 1–5 reduced by 2% each to accommodate Category 6.**

---

## 1) UX Clarity And Flow (18%)
### 10.0 Signals
- First screen is readable and not overloaded.
- Terminal remains primary action surface on desktop and mobile.
- Phase boundaries are obvious and safe stopping points are explicit.
- Resume/checkpoint flow is clear and reliable.
- Running text / ticker animations have a visible pause or dismiss control.
- Side panels can be minimized or dismissed on demand.
- Terminal zone expands dynamically to fill available viewport.

### Deduct If
- Input focus is unreliable.
- Terminal is visually squeezed by guidance panels.
- Learners cannot tell what to do next in under 5 seconds.
- Animation runs without a user-controlled stop option.
- No minimize/collapse control on secondary UI panels.

---

## 2) Pedagogy Compliance (23%)
### 10.0 Signals
- Every tested concept is taught before assessment.
- `pre_step_concept`, `jit`, `bridge`, `teach`, or `concept` coverage is present per assessed step.
- Real-world transfer language is explicit for sim-only commands.
- L.O.L.A. guidance is step-specific, not generic copy.
- J.A.N.E.T. remains professionally cold and distinct from L.O.L.A.
- Spaced repetition callbacks are present — prior concepts surface under new pressure.
- Difficulty slope is consistent within and across phases; no unexplained difficulty spikes.
- Learning objectives are achievable within a single session sitting.

### Deduct If
- Untaught syntax appears in challenges.
- Wrong-answer coaching is flat or non-directional.
- Phase checkpoint contract is missing or inconsistent.
- Spaced repetition is absent in phases 3+.
- A learner can complete a phase without demonstrating the stated competency.

---

## 3) Command Acceptance Robustness (23%)
### 10.0 Signals
- Accept logic supports legitimate command variants.
- Multi-step objectives enforce sequence where intended.
- False negatives are rare in guided and expert playthroughs.
- Hints/diagnostics always match current objective context.
- DIAGNOSE engine covers minimum 3 wrong-answer patterns per step.
- No step ships with only generic "try again" feedback.
- acceptFn logic has been manually verified — no unclosed strings, no regex edge cases.

### Critical Fails
- Correct commands are rejected in normal use.
- Hints reference stale files/commands from another course.
- Progression can be bypassed by clearly incorrect minimal input.
- A step has zero specific DIAGNOSE patterns (generic feedback only).

---

## 4) Performance And Stability (13%)
### 10.0 Signals
- No console errors during full playthrough.
- Interactions are responsive; no major layout thrash.
- Save/resume state is stable across reload.
- No obvious memory leaks from repeated modal/phase interactions.
- Playwright suite (station-delta.spec.ts) passes all assertions — primary flow and advanced QA checks.
- No POST errors in Network tab; webhook firing and confirmed in analytics sheet.

### Deduct If
- Frequent rendering jank.
- State corruption after checkpoint resume.
- Event handlers duplicate after repeated modal opens.
- Playwright suite fails any assertion without documented justification.
- Webhook fires but no row appears in Station Delta Analytics sheet.

---

## 5) Narrative And Voice Quality (13%)
### 10.0 Signals
- Course has distinct identity while staying in Station Delta universe.
- L.O.L.A. is supportive, precise, confidence-building.
- J.A.N.E.T. is memorable, dry, controlled, and professional.
- Tone matches rank and track depth.
- Easter eggs are present, discoverable, and thematically coherent.
- All cross-references to callsigns, ship names, and terminals are course-specific — no prior course artifacts bleed through.

### Deduct If
- Voice drift (generic assistant tone).
- Reused prior-course narrative artifacts break immersion.
- Weak emotional contrast between L.O.L.A. and J.A.N.E.T.
- Easter eggs reference content inconsistent with the Station Delta universe.
- Boss fight (J.A.N.E.T. final assessment) lacks emotional weight or narrative payoff.

---

## 6) Accessibility And Inclusion (10%) *(new)*
### 10.0 Signals
- Terminal output div has `aria-live="polite"` and `role="log"` — screen readers detect updates in real time.
- All interactive elements are keyboard navigable — no mouse-only interactions.
- Color is never the sole indicator of state (error, success, warning all have text or icon accompaniment).
- Font size and contrast meet WCAG 2.1 AA minimum standards.
- No content flashes more than 3 times per second (seizure safety).
- Mobile layout tested on a physical device — no floating windows, no clipped inputs, no broken scroll.
- Touch targets are minimum 44x44px on mobile.
- Input fields receive correct focus on mobile without requiring a manual tap after page interaction.

### Critical Fails
- Terminal output has no `aria-live` attribute — screen readers cannot detect sim responses.
- Interactive elements are unreachable by keyboard alone.
- Mobile layout produces floating or overlapping UI elements that block the terminal.

### Deduct If
- Color contrast fails AA standard on any primary UI element.
- Touch targets are undersized on mobile.
- Focus management is lost after modal open/close cycles.
- Physical device mobile test was not performed (emulator only does not satisfy this requirement).

---

## Required QA Evidence Before Merge
- 1 guided playthrough log (with hints enabled).
- 1 expert playthrough log (minimal hints, speed/robustness focus).
- False negative list (if any) and fix confirmation.
- Playwright suite run log — all tests green or failures documented with resolution.
- Accessibility check log — aria-live confirmed, keyboard nav verified, mobile device tested.
- Analytics confirmation — at least one real session row visible in Station Delta Analytics sheet post-playthrough.
- Final rubric table with category scores and weighted total.

## Release Decision Labels
- **PASS (>=9.5)**: merge and mark live.
- **HOLD (9.0 to 9.49)**: fix high-impact issues, rerun gate.
- **FAIL (<9.0 or critical fail)**: do not merge to live branch.
